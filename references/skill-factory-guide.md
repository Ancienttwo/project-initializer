# Skill Factory Guide

Skill Factory is a built-in project capability generated by `project-initializer`.

## What it does

- watches repo-local workflow artifacts such as `plans/`, `tasks/todo.md`, `tasks/lessons.md`, and `tasks/contracts/`
- proposes reusable user-level skills when repeated workflow or correction patterns appear
- records usage markers, activity history, and explicit feedback for generated skills
- prepares sidecar artifacts so the user-level `autoresearch` skill can optimize them later

## Generated project surfaces

- `.claude/hooks/lib/skill-factory.sh`
- `.claude/hooks/skill-factory-session-end.sh`
- `scripts/skill-factory-check.sh`
- `scripts/skill-factory-create.sh`
- `.claude/skill-factory/` templates

## Generated user-level skill structure

```text
~/.claude/skills/<slug>/
  SKILL.md
  agents/openai.yaml
  .factory/
    meta.json
    rubric.json
    history.jsonl
    feedback.jsonl
    last-used.json
```

Use `history.jsonl` for usage activity. Use `feedback.jsonl` only for explicit corrective feedback.

## Operating model

1. Hooks collect signal and update `.claude/.skill-factory-state.json`.
2. Repeated workflow or lesson patterns produce pending proposals in `~/.claude/.skill-proposals.json`.
3. The user reviews a proposal and creates a skill with `bash scripts/skill-factory-create.sh`.
4. Skill use is marked explicitly with `bash scripts/skill-factory-check.sh --mark-used <slug> --type <workflow|knowledge>`.
5. Post-edit activity accumulates in `~/.claude/skills/<slug>/.factory/history.jsonl` without affecting optimization thresholds.
6. Explicit feedback is recorded with `bash scripts/skill-factory-check.sh --record-feedback <slug> --signal <label>`.
7. When enough explicit feedback exists, prepare an `autoresearch` run from `.factory/rubric.json`, `feedback.jsonl`, and the supporting activity history.

## Safety defaults

- proposal-first
- explicit usage markers
- activity is not feedback
- optimization hints require explicit feedback
- manual promote only
- no automatic overwrite of user-level `SKILL.md`
